Simple Labels in Rails - Part Two

Back in 2009 I wrote an article about creating simple labels in rails applications. In that article I described how to leverage the power of prawn-layout to generate PDFs that consistently generate perfect labels regardless of browser or operating system.

Since then I've been working on method to make generating labels easier in Prawn. After a few days of hacking I'm proud to announce the first version of Prawn/Labels. Prawn/Labels makes generating labels super easy in Prawn. Simply supply Prawn/Labels with an enumerable object, tell it the type of labels you want to generate, and then let Prawn/Labels do all the heavy lifting.

fancy_names = ["Jordan", "Mary", "Sue", "Frank", "Kelly", "Sarah", "Lacy"]


Prawn::Labels.generate("names.pdf", fancy_names, :type => "Avery5160") do |pdf, name|

  # You can write anything to the cell here through the pdf variable which is a Prawn::Document

  pdf.text name

end

As the code example above illustrates most of the work is moved out of your code. The logic of page breaks and grid sizing is all handled by Prawn/Labels. In the block just specify how and what you'd like printed on each of your labels.

To get the code just clone the Prawn/Labels project from github:

git clone git://github.com/jordanbyron/prawn-labels.git

Right now there is only one label type defined (Avery 5160). Label types are defined in lib/prawn/labels/types.yaml and can easily be added. A label definition has the following elements:

#
# prawns' basic unit is PostScript-Point        
# 72 points per inch
#
Avery5160:
  left_margin: 15.822
  right_margin: 15.822

  columns: 3
  rows: 10
  column_gutter: 10
  row_gutter: 0

If you add a new label type and would like to share it just fork the project on github, add your label type to the types.yaml file, then send a pull request and I'll bring in your new type(s).

Rails

Using Prawn/Labels in rails is a snap. Just follow Greg Brown's instructions to get prawn-core installed (which now also includes prawn-layout), http://wiki.github.com/sandal/prawn/using-prawn-in-rails, then do the following:

Add Prawn/Labels to your submodules first:

$ git submodule add git://github.com/jordanbyron/prawn-labels.git vendor/prawn-labels

Now we need to let rails know where the Prawn/Labels files are located. To do this just follow the same steps as you did for prawn:

config.load_paths << "#{RAILS_ROOT}/vendor/prawn-labels/lib"

Almost done. Now we need to add the following line to the config/initializers/prawn.rb file created using Greg's instructions:

require "prawn/labels"

And we are all done. To give it a try running script/console and generating some labels.

exclamations = ["Labels", "In", "Rails", "Oh Yeah!"]

Prawn::Labels.generate("test.pdf", exclamations, :type => "Avery5160") do |pdf, exclaim|
  pdf.text exclaim
end

I hope that this makes generating labels a snap. If you have any comments or questions about the code feel free to get in touch with me through email jordan at ducksoupsoftware.com or on twitter @jordan_byron

Gift List Released!

I am so excited to announce the official release of Gift List! Simply put Gift List is an easy to use gift registry you can setup online and send to your friends and family.

I started working on this project nearly a year ago. Since then I have worked on it on and off until I leveled the entire project this October and began building the Gift List that was released today. Gift List was created from a need to better coordinate gift buying over the holiday season. Prior to Gift List, Kelly and I would email each other and our families our lists. This worked, but it was often difficult coordinating who was buying which gift. We obviously wanted to avoid purchasing the same gift twice and sometimes, because of poor communication, that happened.

I am sure we have all been there when someone opens a gift and you realize it is the same exact thing you purchased for them. Not a good feeling. So Gift List attempts to resolve this issue by allowing a user to create lists and send them out to the same people they would typically send their gift lists. Those friends and family then can log into the Gift List site, see what gifts are available, and which gifts have already been claimed.

Give it a try. Gift List is available now at giftlist.ducksoupsoftware.com. Just signup, create your first list, and send out your invites. It is that easy!
